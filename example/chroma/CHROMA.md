## 0. 这个产品要解决的问题

现有调色板工具的常见问题：需要注册账户、产出物是图片而非代码、不保留历史记录、或者只提供灵感而不提供工程可用的 token。开发者拿到一个颜色后，还需要手动推导色阶、对齐命名规范、验证对比度——这些工作应该自动完成。

本产品的核心假设：

1. **产出物应该直接可用** — 生成的色阶可以零修改粘贴进任何项目，格式覆盖主流工具链
2. **工具本身不该抢主角** — UI 保持中性，生成的色板才是视觉焦点；CHROMA 的界面越透明越好
3. **历史和分享是轻量需求** — localStorage 足够，不需要账户；一条 URL 足以分享，不需要服务器

所有设计决策从这三条出发。如果某个改动和这三条冲突，默认是改动有问题。

---

## 1. 一句话定位

CHROMA 是一个把单个颜色转化为可部署 design token 色阶的工具。

它不是调色灵感板，也不是图片取色器，也不是在线调色工具。

---

## 2. 核心体验

### 2.1 用户打开产品的第一秒

用户首先看到的是：一个颜色输入框（支持 HEX / HSL / OKLCH）和一套实时预览的 11 档色阶色块。页面上没有登录入口，没有引导弹窗，没有"开始使用"按钮。

这一刻的设计目标是：用户立刻知道可以做什么，并且立刻可以开始做。

### 2.2 唯一的核心交互

整个产品只有一个真正的交互：**输入或调整一个颜色，色阶实时更新**。

无需点击"生成"。颜色合法时立即计算，视觉即时反馈。

### 2.3 完成后的出口

用户在确认色阶后，可以：

- 选择导出格式（CSS variables / Tailwind config / JSON），一键复制代码块
- 生成分享链接（色板编码进 URL），发给协作者
- 色板自动存入本地历史，下次打开可继续使用

出口的标准：**代码块可以直接粘贴进项目文件，不需要任何修改**。

## 3. 用户旅程

```
打开页面 → 输入/调整颜色 → 预览色阶 → 确认 → 复制代码 / 分享链接
                                        ↓
                                   历史列表更新（自动）
```

| 步骤 | 用户看到什么 | 用户做什么 | 异常/岔路 |
|------|-------------|-----------|----------|
| 打开页面 | 输入框 + 默认色阶预览 | 可直接修改颜色 | — |
| 输入颜色 | 实时色阶更新 | 调整到满意为止 | 输入非法颜色 → 输入框标红，色阶不更新 |
| 选择格式 | 代码预览面板 | 点击格式标签切换 | — |
| 复制代码 | 按钮变为"已复制" | 粘贴进项目 | — |
| 分享 | 链接复制成功提示 | 发给协作者 | — |
| 查看历史 | 历史色板列表 | 点击恢复某个色板 | localStorage 清空 → 历史为空态 |

## 4. 核心概念与领域模型

### 4.1 领域实体

CHROMA 的核心领域实体：

1. **Palette（色板）** — 一个基色 + 由它生成的 11 档色阶（50, 100, 200, 300, 400, 500, 600, 700, 800, 900, 950），是产品的核心产出物
2. **Stop（色阶档）** — 色板中的单档颜色，有档位号、HEX 值、OKLCH 值、与白/黑背景的对比度比值
3. **ExportFormat（导出格式）** — 支持 CSS variables、Tailwind config、JSON 三种，每种格式的代码字符串由 Palette 实时生成

### 4.2 实体间的关系

```
基色 (HEX/HSL/OKLCH)
  └─ 色阶算法 → Palette
                  ├─ Stop × 11
                  ├─ ExportFormat(CSS)
                  ├─ ExportFormat(Tailwind)
                  └─ ExportFormat(JSON)
```

预设色板数据见 `CHROMA-DATA.md`。UI 文案见 `CHROMA-COPY.md`。

## 5. 产品禁忌

以下特性或设计方向**明确不在产品范围内**：

- 🚫 用户账户和登录
- 🚫 服务端存储任何颜色数据
- 🚫 图片取色功能
- 🚫 "灵感"类功能（情绪标签、颜色故事、配色推荐）
- 🚫 多色板同时展示和对比（每次只聚焦一个色板）
- 🚫 自定义色阶档位数量或命名（始终是 Tailwind 标准 11 档）

> 最后一条尤其重要：允许自定义档位会让产出物失去工具链的标准兼容性，违背假设 1。

---

## 6. 隐私与数据原则

- 用户生成的色板仅存于本地 localStorage，不上传服务器
- 分享链接的全部信息编码在 URL 本身，服务器不存储任何状态
- 没有埋点，没有分析，没有第三方 SDK
